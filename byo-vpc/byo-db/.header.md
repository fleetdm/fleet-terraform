This module deploys the Fleet ECS cluster, ALB, and Fleet services while using an existing database and cache. In the full repo stack it is normally called by `byo-vpc`, but it can also be used independently when those backing services already exist.

# KMS Coverage

This module directly manages optional CMK support for:

* ECS cluster execute-command CloudWatch log groups
* ECS Fargate ephemeral storage

It also passes through nested `byo-ecs` KMS settings for:

* Fleet private-key secret
* Fleet application log groups
* Fleet software installer bucket encryption

# CloudWatch Logs Migration

If you enable KMS for existing ECS cluster or Fleet log groups and need to purge pre-KMS streams, use the repository-root helper:

```bash
DELETE_OLD_STREAMS=false ../../../scripts/cloudwatch_logs_kms_migration.sh <log-group-name> <region>
```
